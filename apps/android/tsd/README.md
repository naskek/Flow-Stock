# FlowStock TSD (PWA)

## Установка PWA (Android Chrome)
1. Откройте `index.html` по HTTPS (через локальный сервер или HTTPS-хостинг).
2. В Chrome нажмите меню ⋮ → "Установить приложение" (или "Добавить на главный экран").
3. Запустите приложение с иконки на домашнем экране.

## Проверка офлайн
- Откройте приложение хотя бы один раз онлайн, чтобы кэш прогрелся.
- Включите режим "В самолёте" / отключите Wi‑Fi и мобильные данные.
- Откройте приложение с иконки — UI и IndexedDB должны работать.

> Важно: для установки и Service Worker нужен HTTPS (или localhost).

## Запуск сервера (FlowStock.Server)
1. На ПК запустите:
   `dotnet run --project apps/windows/FlowStock.Server --launch-profile https`
2. Откройте в браузере:
   `https://<ip-вашего-пк>:7153/index.html`
3. Для dev-сертификата (если браузер ругается):
   `dotnet dev-certs https --trust`

## Запуск на Urovo CT48
1. Скопируйте папку `tsd` на устройство.
2. В файловом менеджере откройте `tsd/index.html`.
3. Выберите Chrome и откройте страницу.

## Сканер (keyboard / intent)
- По умолчанию используется `keyboard wedge` (сканер печатает данные в текущий фокус).
- `Intent`-сканирование поддерживается только через JS-bridge в нативной оболочке (`WebView` / `TWA` / `Capacitor`).
- В чистом `Chrome` / `PWA` broadcast intent недоступен.

### Режимы
- `auto` (по умолчанию): используется `intent`, если доступен bridge, иначе `keyboard`.
- `keyboard`: принудительный режим wedge.
- `intent`: принудительный режим `intent` (если bridge недоступен, будет fallback на `keyboard`).

Способы указать режим:
- В debug-режиме (откройте `index.html?debug=1#/settings`) выберите режим в настройках.
- Через глобальную переменную `window.FLOWSTOCK_SCANNER_MODE = "auto|keyboard|intent"` до загрузки `app.js`.

### Ожидаемый JS-bridge
Нативная оболочка должна предоставить:
`window.FlowStockAndroidBridge.subscribeScans(callback)`

`callback` получает объект:
```
{ value: "460...", symbology: "EAN13", raw: {...}, ts: 1710000000000 }
```

Поддерживаемые ключи для значения: `value`, `data`, `barcode`, `scanData`, `scan_data`, `code`, `text`.

### Keyboard wedge (Chrome/PWA)
- Включите режим **Keyboard**.
- Отключите **Intent output**.
- Рекомендуется суффикс Enter.

## Обновление файлов
- Закройте страницу в Chrome.
- Замените файлы в папке `tsd` на новые (перезапишите).
- Откройте `index.html` снова.

## Работа с операциями
- На главном экране выберите тип операции — откроется список документов этого типа.
- Нажмите "+ Новый", заполните шапку и сканируйте штрихкоды.
- Enter добавляет строку с текущим шагом, Undo отменяет последний скан.
- После заполнения обязательных полей нажмите "Завершить" для статуса READY.
- Под шапкой переключите режим "+1" или "Кол-во..." - второй режим открывает диалог выбора количества после скана.
- Для списаний рядом с полем "Причина" нажмите "Выбрать..." и выберите код (Повреждено, Просрочено, Брак, Проба, Производство, Прочее); без причины завершить документ нельзя.
- Черновики можно удалить кнопкой "🗑️ Удалить черновик" на экране документа, но готовые и экспортированные операции удалить нельзя.
- Если штрихкод не найден в каталоге, появится запрос на создание товара с этим кодом.

## Загрузка данных с ПК
- В настройках нажмите "Загрузить данные с ПК..." и выберите файл `FlowStock_TSD_DATA.json`.
- После импорта появится статус и количество справочников.
- Идентификатор устройства задаётся в `meta.device_id` и назначается при импорте; на ТСД он не редактируется.

## Выбор контрагентов и локаций
- Если справочники загружены, поля контрагентов и локаций выбираются через кнопку "Выбрать...".
- Поиск работает по названию/коду, сверху показываются последние 5 выборов.

## Экспорт JSONL
- В настройках нажмите "Экспорт JSONL".
- Экспортируются только документы со статусом READY, после чего статус становится EXPORTED.
- В настройках доступен экспорт новых товаров в файл `ITEMS_*.jsonl` для импорта на ПК; `item_id` может быть строковым.

## Заказы
- На главном экране нажмите "Заказы", чтобы открыть список заказов.
- Поиск работает по номеру заказа и контрагенту (название/ИНН).
- Откройте заказ, чтобы увидеть статусы и строки заказа.

## Остатки
- Нажмите "Остатки" на главном экране, чтобы перейти в поиск по данным из `FlowStock_TSD_DATA.json`.
- Поле поиска принимает скан или ручной ввод: Enter ищет по штрихкоду через справочник.
- Под полем отображается дата последней выгрузки (`exportedAt`), карточка показывает итог и остатки по локациям.
- Кнопка "Очистить" сбрасывает выбор, стрелка "Назад" вверху возвращает на главный экран.

## Хранение данных
- Настройки сохраняются в IndexedDB в профиле Chrome.
- Для полного сброса настроек очистите данные Chrome.
